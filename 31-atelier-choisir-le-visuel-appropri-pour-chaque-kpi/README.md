# Atelier : Choisir le visuel approprié pour chaque KPI

**Rappel théorique** Un tableau de bord KPI n'est utile que si les données sont présentées de façon lisible et immédiatement interprétable. Le choix du visuel n'est pas une question d'esthétique : c'est une décision analytique. Chaque type de graphique répond à une question précise. Une courbe de tendance (line chart) est idéale pour visualiser l'évolution d'un indicateur dans le temps (ex : trafic organique semaine par semaine). Un graphique en barres convient à la comparaison entre catégories (ex : taux de conversion par canal d'acquisition). Une jauge ou un indicateur de progression (gauge/bullet chart) est pertinent pour afficher l'avancement vers un objectif chiffré (ex : 65 % de l'objectif mensuel de leads atteint). Un entonnoir (funnel chart) est indispensable pour visualiser les étapes d'un parcours et les taux de transformation successifs. Un camembert (pie chart) peut illustrer une répartition, mais devient illisible au-delà de 5 segments. Un tableau de données brutes reste pertinent pour des KPI multidimensionnels nécessitant une lecture précise. L'erreur la plus fréquente est d'utiliser un camembert pour montrer une évolution temporelle, ou une courbe pour comparer des valeurs sans lien chronologique. Un autre piège classique : multiplier les visuels complexes là où un chiffre clé (KPI card) suffirait. Règle d'or : le bon visuel est celui qui permet à un non-expert de comprendre le message en moins de 5 secondes.
