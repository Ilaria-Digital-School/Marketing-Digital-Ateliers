# Atelier : Identifier leading indicators pour objectif donné

**Rappel théorique : Leading vs Lagging KPI** Avant d'entrer dans le vif du sujet, rappelons l'essentiel. Un lagging indicator (indicateur retardé) mesure un résultat passé : le chiffre d'affaires du mois, le taux de churn, le nombre de clients acquis. Il est précieux pour évaluer la performance, mais inutile pour l'anticiper ou la piloter en temps réel. Un leading indicator (indicateur avancé) mesure une action ou un comportement qui précède et prédit le résultat futur. C'est lui qui vous permet d'agir avant qu'il ne soit trop tard. Le mental model à retenir : les leading indicators sont aux lagging indicators ce que la météo est à la récolte. Vous ne pouvez pas changer la récolte d'hier, mais surveiller la météo vous permet de prendre les bonnes décisions aujourd'hui. Dans un entonnoir AARRR, chaque étape génère des leading indicators pour l'étape suivante : le nombre de visiteurs uniques (Acquisition) est un leading indicator du nombre d'inscriptions (Activation). Erreurs fréquentes à éviter : confondre un leading indicator avec une action interne (ex. : 'publier 3 articles par semaine' est une tâche, pas un KPI) ; choisir des leading indicators non corrélés à l'objectif final ; en sélectionner trop, ce qui dilue l'attention et l'effort de suivi. Mini-exemple : Objectif lagging — augmenter le taux de conversion d'essai gratuit vers abonnement payant de 8 % à 14 % en 3 mois.
